# Problem Overview
You have a dataset where each output was generated by either **Model A** or **Model B**, but you don't know which one.
* **Given:** The inputs, the **intermediate embeddings** from both models, and the final observed outputs.
* **Missing:** The final layer weights that connect the embeddings to the outputs.
**The Task:** Predict which model (A or B) generated each row by figuring out which model's embedding best explains the final output.
# EDA
The [EDA](https://en.wikipedia.org/wiki/Exploratory_data_analysis) proves that a single global model is destined to fail because it ignores the latent switching between two distinct regimes.
- Plot 2 (Bimodal Residuals): The non-Gaussian, "lumpy" error distribution confirms the data is a mixture of two different functional laws.
- Plot 3 (Variance Imbalance): Model A (blue) is a high-capacity feature extractor, while Model B (red) is relatively "quiet" or low-rank; a standard model will overfit to A and treat B as noise.

# Why EM?
[EM](https://www.geeksforgeeks.org/machine-learning/ml-expectation-maximization-algorithm/) is the first tool to grab whenever you need to group data by hidden module rather than by observed similarity.
The common thread is always:
1. You believe there are K latent regimes (clusters, experts, heads, states, segments).
2. Inside each regime the data obey the same parametric law (linear, GLM, Gaussian, Bernoulli, Poisson, low-rank, …) differing only in parameter values.
3. You do not know which regime generated each point.
4. You do have the sufficient statistics (embeddings, covariates, side information) that feed those laws.

# How does EM work?
### 1. Initialization
First, we create random guesses for the final "hidden" layers ( weights) for both models.
```python
import numpy as np

# Randomly initialize the missing final weights for A and B
W_A = np.random.randn(10, 5)
W_B = np.random.randn(10, 5)

```
### 2. The E-Step: Assigning Credit
We use the intermediary embeddings to see which model's current weights better predict the observed output .
```python
# Predict Y using the intermediary embeddings and current weights
dist_A = np.sum((Y - (emb_A @ W_A))**2, axis=1)
dist_B = np.sum((Y - (emb_B @ W_B))**2, axis=1)

# Calculate "Responsibilities" (Probabilities)
# If dist_A is small, prob_A will be close to 1
prob_A = np.exp(-dist_A) / (np.exp(-dist_A) + np.exp(-dist_B))
prob_B = 1 - prob_A
```
### 3. The M-Step: Updating the Weights
We update the final layers by solving a "Weighted Least Squares" problem. Each model only learns from the rows where it had a high probability in the previous step.

```python
# Update Model A weights using its assigned rows
W_A = np.linalg.solve(emb_A.T @ (prob_A[:, None] * emb_A), 
                      emb_A.T @ (prob_A[:, None] * Y))

# Update Model B weights using its assigned rows
W_B = np.linalg.solve(emb_B.T @ (prob_B[:, None] * emb_B), 
                      emb_B.T @ (prob_B[:, None] * Y))
```
### 4. Convergence and Final Prediction
You wrap the steps above in a loop. Once the weights stop changing (or it has converged), you make your final choice for each row.

```python
# Final decision: 1 for Model A, 0 for Model B
final_predictions = (prob_A > 0.5).astype(int)
```
